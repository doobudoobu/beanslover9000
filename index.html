Anthropic's API does not allow direct browser-to-API calls from arbitrary origins, so the request fails before it even reaches the server. Here is why: The math is very low volume Each time you send a message, it makes exactly one API call. A typical exchange (your message + the conversation history + the system prompt) is probably 500-1500 tokens in, and the response is maybe 200-400 tokens out. At Sonnet's current pricing that is fractions of a cent per message. A full 20-message session with a few photo uploads might cost $0.05-0.15 total. A single uploaded photo adds roughly 1,000-2,000 tokens to the request depending on its size. Still not significant at personal use scale, but worth knowing if you ever left it running in a loop somehow. Where costs can actually creep up The conversation history is re-sent with every message. That is necessary for memory to work, but it means a very long session (50+ messages) starts sending a large payload each turn. The app as built has no history truncation, so an unusually long session could get moderately expensive near the end. A simple fix would be to cap history at the last 20 messages.
